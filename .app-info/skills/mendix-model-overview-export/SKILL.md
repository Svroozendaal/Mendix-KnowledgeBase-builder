# MENDIX MODEL OVERVIEW EXPORT

## Purpose

Use this skill when implementing or extending the full-model overview exporter (not diff export).

## Required inputs

1. `working-dump.json` generated by `mx dump-mpr`.
2. `studio-pro-extension-csharp/Processing/ModelDiff/MendixModelOverviewParser.cs`.
3. `studio-pro-extension-csharp/Processing/Services/AutoCommitMessageModelOverviewService.cs`.
4. `studio-pro-extension-csharp/Docs/MODEL_OVERVIEW_EXPORT_CONTRACT.md`.
5. Diff parser references (for reuse and compatibility guardrails):
   - `.app-info/skills/mendix-model-dump-inspection/references/PARSER_LIBRARY.md`
   - `.app-info/skills/mendix-model-dump-inspection/references/RULE_LIBRARY.md`

## Workflow

1. Parse one working dump snapshot (no head comparison).
2. Build object and resource indexes by `$ID`.
3. Inventory domain model elements by module:
   - entities (attributes + persistability + generalisation)
   - associations (parent/child/cardinality)
   - enumerations (values)
4. Inventory flows by module:
   - microflows
   - nanoflows
   - rules
   - workflows
5. Determine flow execution order using graph traversal:
   - start nodes from `StartEvent` (fallback: zero-incoming nodes)
   - edges from `SequenceFlow(origin,destination)`
   - deterministic edge sorting
   - branch and loop-aware pseudocode output
6. Detect flow-to-flow calls and build a call graph.
7. Export:
   - app overview JSON + pseudocode
   - module overview JSON + pseudocode
   - run manifest/index

## Compatibility guardrails

1. Do not change diff parser behaviour unless explicitly requested.
2. Reuse existing parser conventions from diff parsing when possible.
3. Keep output deterministic and stable for downstream parsers/AI.
4. Additive contract changes only; avoid breaking existing consumers.
