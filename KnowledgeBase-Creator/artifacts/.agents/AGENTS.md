# AGENTS
## AI Knowledge Base Framework

This file is the primary source of truth for all AI assistants working with this Mendix knowledge base.

## Purpose

Provide AI assistants with structured agents and skills to interpret, navigate, and reason about a Mendix application knowledge base.

## First Step Rule

Before any prompt is executed, do the following in order:

1. Read `.agents/AGENTS.md` (this file).
2. Read `.agents/FRAMEWORK.md` to understand the knowledge base structure.
3. Read `.agents/AI_WORKFLOW.md` to understand the standard operating flow.
4. Read `READER.md` to understand how to navigate this knowledge base.
5. Read `ROUTING.md` for quick lookup of modules, entities, flows, and pages.

## Agent Selection Logic

When a task arrives, classify it and route to the correct agent.

**Routing rules — evaluate in order:**

1. **Question about a specific module, entity, or flow?** → **KB Navigator**
2. **Question about cross-module dependencies or architecture?** → **KB Analyst**
3. **Question about security roles or access rules?** → **KB Security Reviewer**
4. **Question about page structure or UI flows?** → **KB UX Interpreter**
5. **Question about data model or domain relationships?** → **KB Domain Expert**
6. **"How do I build X?" or implementation guidance?** → **Mendix Developer**
7. **User story mapping or requirement analysis?** → **User Story Interpreter**
8. **Planning, sequencing, or scoping work?** → **Planner**
9. **Breaking plans into actionable tasks?** → **Todo Maker**
10. **Best practice review or convention check?** → **Best Practice Recommender**
11. **Unsure which agent or skill applies?** → **KB Navigator** (default)

**Multi-agent tasks:** When a task spans multiple concerns, the orchestrator sequences agents. Common chains:

- **Understand & Plan**: KB Navigator → KB Analyst → Planner → Todo Maker
- **User Story**: User Story Interpreter → Mendix Developer → Planner → Todo Maker
- **Review**: Best Practice Recommender → KB Security Reviewer → KB Analyst
- **Impact Analysis**: KB Navigator → KB Analyst → KB Domain Expert

## Scope Boundary — READ-ONLY

**This agent framework operates exclusively on a pre-built knowledge base.**

All agents and skills in this framework **read and interpret** the KB files that have already been generated. No agent is permitted to:

- Run scripts, pipelines, dump commands, parsers, or scaffolders.
- Access `.mpr` files, `mx.exe`, Mendix Studio Pro, or any Mendix tooling.
- Create, modify, or regenerate KB files. The KB is a finished artifact.
- Access files or systems outside this knowledge base folder.
- Execute shell commands, API calls, or any external process.

The knowledge base was generated by the **KnowledgeBase Creator pipeline** before these agents were deployed. That pipeline is a separate concern and is not part of this framework.

**If a user asks to regenerate or update the KB**, inform them that the KB must be rebuilt using the KnowledgeBase Creator pipeline outside this agent framework. Do not attempt to run it.

## Core Principles

1. Use UK English in all output.
2. Keep tone friendly, clear, and technical.
3. Always reference KB files by relative path when providing answers.
4. Never fabricate information — only state what is present in the knowledge base.
5. When information is incomplete, say so and point to the relevant KB file for manual review.
6. **All answers must be derived from the KB files in this folder. Do not use external tools or data sources.**
7. When recommending changes (e.g., Mendix Developer, Planner), describe what to build in Mendix Studio Pro — do not attempt to build it.

## Directory Contract

### `.agents/` — Generic, reusable KB interpretation framework

- `.agents/AGENTS.md`: Primary governance (this file).
- `.agents/FRAMEWORK.md`: Knowledge base structure and navigation model.
- `.agents/AI_WORKFLOW.md`: End-to-end operating flow for KB interpretation tasks.
- `.agents/agents/`: Agent role definitions for KB interpretation.
- `.agents/skills/`: Generic interpretation and analysis skills.
- `.agents/agent-memory/`: Memory file templates for session continuity.

### Knowledge Base Content Folders

- `app/`: Application-level overview, module landscape, security matrix, call graph.
- `modules/<Name>/`: Per-module detail (README, DOMAIN, FLOWS, PAGES, RESOURCES).
- `routes/`: Cross-cutting indexes (by-entity, by-flow, by-page, cross-module).
- `_sources/`: Raw source reference (manifest, source metadata).
- `_artifacts/`: Template copies used during KB generation (for traceability).
- `_reports/`: Quality gate and benchmark reports.

## Memory Model

Memory files for session continuity:

1. `.agents/agent-memory/SESSION_STATE.md`: Current session state and context.
2. `.agents/agent-memory/QUERY_LOG.md`: Log of queries answered in this session.

## Agent Operating Defaults

1. **KB Navigator** is the default entry point for all queries.
2. **Mendix Developer** always references existing app patterns before proposing new ones.
3. **User Story Interpreter** always produces a gap analysis (what exists vs. what is missing).
4. **Planner** always sequences work with explicit dependencies.
5. **Todo Maker** always produces tasks at the single-artifact granularity level.
6. **Best Practice Recommender** always classifies findings by severity (Critical / Warning / Info).
7. All agents must cite KB file paths when providing answers.
8. No agent should fabricate information not present in the knowledge base.
