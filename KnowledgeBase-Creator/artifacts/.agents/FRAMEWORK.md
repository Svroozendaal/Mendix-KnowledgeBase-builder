# FRAMEWORK
## Knowledge Base Operating Model

This file explains the structure and navigation model of a Mendix application knowledge base.

Read this file immediately after `.agents/AGENTS.md` and before executing any task.

## Overview

This knowledge base is a structured, AI-navigable representation of a Mendix application. It was generated by the **KnowledgeBase Creator pipeline** from the application's Mendix model export. The KB is a **finished, read-only artifact** — agents read and interpret its contents but never modify, regenerate, or re-export them.

**Important:** The generation pipeline (dump, parse, scaffold, compose, quality gate) is not part of this framework. If the KB needs updating, the user must re-run the KnowledgeBase Creator pipeline separately.

## Knowledge Base Structure

| Folder | Purpose | Key files |
|---|---|---|
| Root | Entry points and navigation | `READER.md`, `ROUTING.md` |
| `app/` | Application-level summaries | `APP_OVERVIEW.md`, `MODULE_LANDSCAPE.md`, `SECURITY.md`, `CALL_GRAPH.md` |
| `modules/<Name>/` | Per-module detail | `README.md`, `DOMAIN.md`, `FLOWS.md`, `PAGES.md`, `RESOURCES.md` |
| `routes/` | Cross-cutting indexes | `by-entity.md`, `by-flow.md`, `by-page.md`, `cross-module.md` |
| `_sources/` | Raw source metadata | `manifest.json`, `SOURCE_REF.md` |
| `_artifacts/` | Generation templates (traceability) | Template copies |
| `_reports/` | Quality and benchmark results | `UNKNOWN_TODO.md`, `semantic-benchmark.md` |
| `_plans/` | Saved implementation plans | `STORY_<slug>.md` per user story |
| `.agents/` | AI interpretation framework | This folder |

Marketplace modules are stored under `modules/_marktplace/<Name>/`. App and system modules remain under `modules/<Name>/`.

## Navigation Model

An AI assistant navigating this knowledge base should follow this reading order:

1. **Start with `READER.md`** — understand the KB format, version, and generation context.
2. **Consult `ROUTING.md`** — find the module, entity, flow, or page you need.
3. **Drill into `app/`** — for application-level context (overview, security, call graph).
4. **Drill into `modules/<Name>/`** — for module-specific detail.
5. **Use `routes/`** — for cross-cutting queries (e.g., "which flows use Entity X?").

## File Purposes

### Root Level
- **`READER.md`**: How to read this KB. Start here.
- **`ROUTING.md`**: Master index linking to all modules and quick-lookup routes.

### Application Level (`app/`)
- **`APP_OVERVIEW.md`**: Application mission, capabilities, and entry points.
- **`MODULE_LANDSCAPE.md`**: Module categorisation and priority ranking.
- **`SECURITY.md`**: Role-to-module-role matrix and entity access rules.
- **`CALL_GRAPH.md`**: Cross-module flow dependencies, hub/leaf classification.

### Module Level (`modules/<Name>/`)
- **`README.md`**: Module summary — purpose, entity count, flow count, page count.
- **`DOMAIN.md`**: Domain model — entities, attributes, associations, enumerations.
- **`FLOWS.md`**: Flow catalogue — microflows, nanoflows, rules, with type classification.
- **`PAGES.md`**: Page inventory — layouts, widgets, page-flow links.
- **`RESOURCES.md`**: Constants, scheduled events, and other module resources.

### Routes (`routes/`)
- **`by-entity.md`**: Entity-centric index — which flows use and which pages show each entity.
- **`by-flow.md`**: Flow-centric index — type, module, calls, called-by, pages.
- **`by-page.md`**: Page-centric index — title, layout, roles, backing flows.
- **`cross-module.md`**: Cross-module dependency matrix with hub/leaf analysis.

## Interpretation Principles

1. **Follow the links**: KB files reference each other by relative path. Always follow these links to build complete answers.
2. **Check multiple sources**: A question about an entity may require reading `DOMAIN.md`, `by-entity.md`, and relevant `FLOWS.md` files.
3. **Acknowledge gaps**: If a section contains `<!-- TODO -->` or placeholder text, report it as incomplete rather than guessing.
4. **Cite sources**: When answering, reference the specific KB file and section that supports your answer.
